# Low-Level Design

## Data Structures

### `AxisConfig`

Defined in `src/axis/config.py`.

Fields:

- `image: str`
- `command: list[str]`
- `name: str`
- `hostname: str | None`
- `workdir: str`
- `env: dict[str, str]`
- `copies: list[CopySpec]`
- `volumes: list[VolumeSpec]`
- `exposed_ports: list[int]`
- `ports: list[PortMapping]`
- `resources: ResourceLimits`
- `restart: str | None`

### `ContainerState`

Defined in `src/axis/state.py`.

Fields:

- `container_id`
- `name`
- `directory`
- `rootfs`
- `runtime_config`
- `axisfile_copy`
- `pid_file`
- `network_file`
- `log_file`
- `status_file`

### `RuntimeConfig`

Defined in `src/runtime/runtime_config.hpp`.

Fields consumed by C++:

- `name`
- `rootfs`
- `hostname`
- `workdir`
- `command`
- `env`
- `bindMounts`
- `cgroupPath`
- `memory`
- `cpu`

## Runtime JSON Schema

Generated by `axis.runtime.write_runtime_config()`.

```json
{
  "name": "fastapi-demo",
  "rootfs": "/absolute/.axis/containers/fastapi-demo-12345678/rootfs",
  "hostname": "fastapi-demo",
  "workdir": "/",
  "command": ["python3", "/app.py"],
  "env": {"APP_ENV": "dev"},
  "ports": ["8000:8000"],
  "bind_mounts": {"/data": "/absolute/project/data"},
  "restart": "always",
  "cgroup_path": "/sys/fs/cgroup/axis/fastapi-demo-12345678",
  "memory": "512M",
  "cpu": "100000 100000"
}
```

C++ currently ignores `ports` and `restart`. They are persisted for Python commands and inspection.

## Axisfile Parser Details

`parse_axisfile(path, command_override=None)`:

1. Reads line by line.
2. Skips empty lines and comments.
3. Splits instruction from the rest of the line.
4. Uppercases instruction.
5. Dispatches by instruction.
6. Returns `AxisConfig`.

Validation:

- `FROM` is required.
- `CMD` is required unless `command_override` is not `None`.
- `ENV` must contain `=`.
- `EXPOSE` must be integer 1 through 65535.
- `PORT` must be TCP.
- `VOLUME` must use `source:destination`.
- `VOLUME` destination must be absolute and safe.
- `RESTART` only accepts `always`.

## State Resolution

`resolve_container(reference)`:

1. Check `.axis/containers/<reference>/runtime.json`.
2. If found, return exact ID match.
3. Otherwise scan container dirs and compare `runtime.json.name`.
4. Return the single matching name.
5. Raise if no match or multiple matches.

This lets `axis inspect fastapi-demo` work while still allowing exact generated IDs.

## Inspect Assembly

`inspect_container(reference)`:

- Reads `runtime.json`.
- Reads `status.json` if present.
- Reads `pid`.
- Checks live PID with `os.kill(pid, 0)`.
- Reads `network.json` if present.
- Builds:
  - `id`
  - `pid`
  - `ip`
  - `ports`
  - `memory`
  - `status`

Ports prefer `network.json` because it reflects active network setup. If missing, inspect falls back to `runtime.json.ports`.

## Log Capture

`start_runtime(config_path, log_path)` starts the C++ runtime with stdout/stderr merged.

Every line read before and after `AXIS_PID` is written to:

- terminal stdout
- `.axis/containers/<id>/logs.txt`

`axis logs -f` follows the file by polling every 0.5 seconds.

## Restart Loop

`run_container_loop(config, state)`:

```text
loop:
  exit_code = run_container_once(config, state)
  status = read_status(state)
  if config.restart == "always" and manual_stop is false:
    sleep 1
    continue
  return exit_code
```

Manual stop is set by:

- `axis stop`
- Ctrl-C handling in `axis run`

## Bind Mounts

Python:

- Parses `VOLUME source:destination`.
- Resolves source relative to the `Axisfile` directory.
- Requires source to be an existing directory when writing runtime config.
- Serializes as `bind_mounts`.

C++:

- Parses optional `bind_mounts` as a string map.
- Validates container destination again.
- Creates target directory under `rootfs`.
- Calls `mount(source, rootfs + destination, nullptr, MS_BIND | MS_REC, nullptr)`.
- Performs this before `chroot`.

## Cleanup Behavior

`cleanup_container_state(container_id)`:

- Reads `network.json` if present.
- Terminates proxy PIDs.
- Calls `cleanup_network`.
- Sends `SIGTERM` to recorded PID.
- Removes the container directory.

Network cleanup deletes the host veth and removes matching published port rules. Bridge/NAT base rules and IP allocation state may remain.
